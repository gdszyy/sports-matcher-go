# 异构体育赛事匹配算法：优化测试报告

**作者**：Manus AI
**日期**：2026-04-17
**状态**：已完成并验证

## 1. 测试概述

本报告记录了针对 `sports-matcher-go` 仓库中异构体育赛事匹配算法（SR↔TS / LS↔TS）最新优化成果的系统性测试。测试基于离线 Mock 数据集，覆盖了联赛匹配、比赛匹配、球队名称相似度计算以及特征提取与否决等核心模块，旨在验证优化建议文档中提出的各项改进是否达到预期效果。

测试套件共包含 40 个单元测试与集成测试，涵盖了正常匹配、边界场景以及各类负向约束场景。

## 2. 测试场景与结果分析

### 2.1 联赛匹配（League Match）

联赛匹配是整个算法链路的基础，其准确性直接影响后续比赛和球队的匹配质量。本次测试重点验证了强约束否决机制和负向特征惩罚的有效性。

| 测试场景 | 验证目标 | 预期结果 | 实际结果 | 状态 |
|---------|---------|---------|---------|------|
| **已知映射命中** | 验证 `KnownLeagueMap` 是否具有最高优先级 | 直接返回置信度 1.0 | 置信度 1.0，规则 `RuleLeagueKnown` | ✅ PASS |
| **名称高置信度命中** | 验证无已知映射时，名称相似度是否能正确匹配 | 返回高置信度匹配 | 匹配成功，规则 `RuleLeagueNameHi` | ✅ PASS |
| **地理别名词典** | 验证 "USA" 与 "United States" 是否被识别为同一国家 | 获得地理加分 | 匹配成功，获得地理加分 | ✅ PASS |
| **年龄段差异惩罚** | 验证 "U19" 与 "U21" 是否被大幅降分 | 惩罚系数 ≤ 0.35，最终得分 < 0.55 | 惩罚系数 0.30，最终得分 0.00 | ✅ PASS |
| **预备队强约束否决** | 验证 "Reserves" 是否被禁止匹配主联赛 | 触发强约束否决 | 成功否决 | ✅ PASS |
| **州联赛强约束否决** | 验证 "Campeonato Paulista" 是否被禁止匹配全国联赛 | 触发强约束否决 | 成功否决 | ✅ PASS |
| **多语言层级数字** | 验证 "Segunda Division" 等多语言序数词是否被正确提取 | 正确提取层级数字 | 成功提取（如 Segunda → 2） | ✅ PASS |
| **跨国误匹配防护** | 验证 "Libya" 与 "Laos" 是否被地理否决 | 触发地理否决 | 成功否决 | ✅ PASS |
| **性别强约束否决** | 验证 "Women" 与 "Men" 是否被禁止匹配 | 触发强约束否决 | 成功否决 | ✅ PASS |

**关键修复记录**：在初始测试中，发现 "Liga 2 Peru" 与 "Liga 1 Peru" 未被层级数字否决。经诊断，原逻辑中层级否决仅在 `med/low` 置信度时生效，而别名索引导致两者初始相似度极高（`hi` 置信度），从而绕过了否决。**修复方案**：将层级数字冲突提升为全局强约束，在所有置信度级别下均生效。修复后测试通过。

### 2.2 比赛匹配（Event Match）

比赛匹配测试验证了高斯衰减连续模糊时间窗口以及不同匹配策略的有效性。

| 测试场景 | 验证目标 | 预期结果 | 实际结果 | 状态 |
|---------|---------|---------|---------|------|
| **标准时间窗口（策略1）** | 验证时差 < 6h 且名称相似度高时的匹配 | 匹配成功，规则 `RuleEventL1` | 匹配成功，规则 `RuleEventL1` | ✅ PASS |
| **宽松时间窗口（策略2）** | 验证时差 8h 且名称完全一致时的匹配 | 匹配成功，规则 `RuleEventL2` | 匹配成功，规则 `RuleEventL2` | ✅ PASS |
| **同日 UTC 匹配（策略3）** | 验证跨日时区（时差 22h）但同日（UTC）的匹配 | 匹配成功，规则 `RuleEventL3` | 匹配成功，规则 `RuleEventL3` | ✅ PASS |
| **主客场反转处理** | 验证主客场反转时的引擎行为 | 允许匹配（覆盖率优先） | 匹配成功，记录结果供分析 | ✅ PASS |

**设计说明**：关于主客场反转，现有引擎在 `tryMatchLevel` 中使用 `max(正向, 反向)` 计算名称相似度。这是已知的设计权衡，旨在兼容部分数据源主客场标注不一致的情况，以提升整体覆盖率。

### 2.3 球队名称相似度（Team Name Similarity）

球队名称相似度测试验证了 Jaro-Winkler 融合算法对前缀相似名称的提升效果，以及对明显不同名称的区分能力。

| 测试场景 | 验证目标 | 预期结果 | 实际结果 | 状态 |
|---------|---------|---------|---------|------|
| **Jaro-Winkler 提升** | 验证 "Chaidari" vs "Chaidari AO" 等前缀相似名称 | 相似度显著提升（≥ 0.70） | 相似度 0.945 | ✅ PASS |
| **明显不同名称区分** | 验证 "Arsenal" vs "Juventus" 等无关名称 | 相似度保持在低位（≤ 0.40） | 相似度 0.000 | ✅ PASS |

**关键修复记录**：在初始测试中，发现 "Arsenal" 与 "Juventus" 的 Jaro-Winkler 相似度高达 0.512。这是因为 Jaro-Winkler 是字符级匹配，对较短的单词容易产生假阳性。**修复方案**：在 `teamNameSimilarity` 中对 Jaro-Winkler 引入最低采用阈值（`jwMinThreshold = 0.75`），仅当 JW 得分足够高时才采纳，有效过滤了假阳性。修复后测试通过。

## 3. 结论与建议

本次优化测试结果表明，新引入的**六维强约束一票否决**、**负向特征惩罚**、**地理别名词典**以及**Jaro-Winkler 融合**等机制均已按预期工作。特别是在解决联赛名称泛化误吸附（如 U19 匹配 U21、预备队匹配主联赛、不同层级联赛互匹）方面效果显著。

**后续建议**：
1. **持续扩充词典**：随着接入更多冷门联赛，建议持续维护和扩充 `docs/league_guard_keywords.json` 中的多语言关键词和地理别名。
2. **监控主客场反转**：虽然当前允许主客场反转匹配以提升覆盖率，但建议在生产环境中监控此类匹配的比例，若发现大量误匹配，可考虑引入更严格的惩罚机制。
3. **观察 JW 阈值**：当前设定的 Jaro-Winkler 最低采用阈值（0.75）在测试集中表现良好，建议在真实数据中观察其对召回率和准确率的平衡效果。
